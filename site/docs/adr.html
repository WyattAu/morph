Architecture Decision Records

Key design decisions for the Morph project

ADRs capture significant architectural decisions, their context, rationale, and consequences. All records live under .specs/02_adrs/ in the repository.

Toolchain and Dependencies

ADR-001: Lean 4.28.0-rc1 Migration

Decision to migrate the Lean toolchain to version 4.28.0-rc1, including rationale for the upgrade path and compatibility considerations.

ADR-003: Dependency Version Alignment

Policy for aligning dependency versions across the project to ensure reproducible builds and consistent behavior.

ADR-003: Lean 4 and mathlib4

Decision on the relationship between Lean 4 and mathlib4, including version compatibility and dependency management strategy.

Module Organization

ADR-001: Three-File Module Pattern

Establishes the three-file module pattern (types, rules, proofs) for organizing Lean specification modules.

ADR-004: Three-File Module Pattern (revised)

Revised version of the three-file module pattern decision, incorporating lessons learned from initial adoption.

ADR-005: Domain-Based Module Organization

Decision to organize modules by domain (type, concurrency, memory, etc.) rather than by technical layer.

Code Quality

ADR-002: Zero-Tolerance Commented Code

Policy of zero tolerance for commented-out code in the repository. All dead code must be removed, not commented.

ADR-002: ProofWidgets Removal

Decision to remove ProofWidgets dependency and rationale for simplifying the proof toolchain.

ADR-006: Complete Proof Requirement

Requirement that all specification properties must have complete proofs. No sorry or partial proofs in production modules.

Build and Infrastructure

ADR-004: Lake Build System

Decision to use Lake as the primary build system for Lean 4, including configuration approach and dependency management.

ADR-007: CI/CD Integration

CI/CD pipeline design: GitHub Actions workflows, pre-commit hooks, build verification, and deployment strategy.
