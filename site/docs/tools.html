spec-tools

Python-based specification validation and linting

spec-tools provides automated validation for all 60+ specification modules. It enforces format consistency, checks cross-references, validates link integrity, and reports coverage metrics. 636 tests with 88% coverage.

Installation

# Install spec-tools as an editable package
pip install -e scripts/

Usage

# Validate all specification modules
spec-tools validate spec/

# Lint specification files for style and format
spec-tools lint spec/

# Run the full test suite
pytest scripts/

Capabilities

Format Checking

Validates that each specification file follows the required frontmatter format, section structure, and markdown conventions.

Linting

Checks for common issues: trailing whitespace, line length, heading hierarchy, missing sections, and style violations.

Cross-Reference Validation

Verifies that all internal cross-references between specification modules resolve correctly. Detects dangling references.

Link Integrity

Checks that all external links (GitHub URLs, documentation references) are valid and reachable.

Structural Analysis

Analyzes the dependency graph between specification modules. Reports orphaned files, circular references, and coverage gaps.

Coverage Reporting

Generates coverage metrics showing which specification areas are complete, partial, or missing documentation.

CI Integration

Pre-commit Hooks

spec-tools runs as part of pre-commit hooks, enforcing build, lint, format, and type-check gates before every commit.

GitHub Actions

Spec validation runs in CI on every push and PR. Separate workflow for nightly Nix-based builds with full environment.
