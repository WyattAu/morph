Morph

A formally verified specification for an agentic programming language

Morph provides a comprehensive mathematical foundation for building intelligent, concurrent, and type-safe systems. The entire language specification is verified in Lean 4 with mathlib4, yielding machine-checked proofs of type soundness, progress, and preservation.

328 Lean jobs

Overview

Type System

Dependent types with de Bruijn indices, function types, product types, recursive types, and polymorphic operations. Type soundness verified via Progress and Preservation theorems.

Operational Semantics

Small-step operational semantics covering arithmetic, comparison, logic, bitwise operations, lambda calculus, let bindings, conditionals, for loops, blocks, and function application with eager evaluation.

Formal Verification

Machine-checked proofs in Lean 4 v4.27.0 using mathlib4, aesop, and batteries. Type soundness (Progress + Preservation) proven for the core calculus. Substitution lemmas capture-avoidance by construction.

Agent Programming

Designed for the post-text era of computing. The specification covers process algebra for agent communication, green thread scheduling, capability-based security, and information flow analysis.

Verification Status

Progress Theorem

Proven

Every well-typed expression is either a value or can take a step.

Preservation Theorem

Partial

Type preservation under substitution proven. Simultaneous substitution and for-loop execution have documented gaps (10 sorries across Preservation.lean and Lemmas.lean).

Type System Spec

Complete

HasType and HasTypeAll inductive families with 19 constructors covering all expression forms.

Tools and Infrastructure

Lean 4 v4.27.0

Theorem prover and specification language. Build system via Lake. Proof automation with aesop and batteries. Standard library: mathlib4.

spec-tools (Python)

636 tests validating specification quality: format checking, linting, cross-reference validation, link checking, structural analysis, and coverage reporting. 88% test coverage.

CI/CD

GitHub Actions with four workflows: Lean build verification, specification validation, nightly Nix-based builds, and GitHub Pages deployment. Pre-commit hooks enforce build, lint, format, and type-check gates.

Getting Started

Prerequisites

Lean 4 v4.27.0 (managed by elan), Lake build system.

Build

# Install toolchain (elan handles Lean version)
curl https://raw.githubusercontent.com/leanprover/elan/master/elan-init.sh -sSf | sh

# Build the project
lake build

# Run Lean tests
lake build Morph.Tests

Python spec-tools

# Install and run validation
pip install -e scripts/
spec-tools validate spec/
spec-tools lint spec/
